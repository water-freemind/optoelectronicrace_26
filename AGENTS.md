Repository type: embedded C (TI / Code Composer Studio)
- Single CCS/Eclipse managed-project located at the repo root. Project name: optoelectronicrace_26
- Target MCU: MSPM0G3507 (Cortex-M0+). Toolchain: TI ARM Clang (TICLANG_4.0.4.LTS) per .cproject.

What an agent would otherwise miss
- This is a CCS (Code Composer Studio) managed build: do not assume npm/gradle/autotools. The .project and .cproject are authoritative.
- The Debug/ directory contains generated makefiles, object lists and link-info (Debug/*.mk, *_linkInfo.xml). Those files are produced by CCS and should not be edited manually.

How to build (short)
- Preferred: open the project in Code Composer Studio and use its Build command (this ensures CCS sets all toolchain and SDK variables).
- Alternative (advanced): you can run GNU make using the generated makefiles, but only after you have installed CCS + the MSPM0 SDK and ensured the CCS toolchain and SDK environment variables (COM_TI_MSPM0_SDK_INSTALL_DIR, CG_TOOL_ROOT, etc.) match those in .cproject. The linker command in Debug/optoelectronicrace_26_linkInfo.xml shows the exact compiler/linker binary and flags used by CCS.
- Don't attempt to build on CI or a machine that doesn't have CCS/MSP SDK installed — paths in the project are absolute (D:/CCS, D:/MSP-SDK) and the toolchain is proprietary.

Entry points & source layout (what to edit)
- Primary application: main.c at repo root.
- Primary code directories: BSP/, Drive/OLED_Drv/, Menu/ (see Debug/sources.mk SUBDIRS). Edit sources in these folders; they compile into Debug/optoelectronicrace_26.out.

Files to avoid changing blindly
- .project and .cproject define the CCS project settings and include paths; change only if you know CCS/Eclipse managed-build implications.
- Everything under Debug/ is generated by the build — don't hand-edit generated make fragments, sources.mk, objects.mk, or the _linkInfo.xml unless you explicitly intend to change build outputs.

Toolchain / flags / device-specific notes
- cproject forces MCPU=cortex-m0plus, OUTPUT_FORMAT=ELF, and uses device configuration DEVICE_CONFIGURATION_ID=Cortex M.MSPM0G3507. Do not change CPU/ABI/compiler selection without rebuilding in CCS.
- Linker output and entrypoint in Debug/*_linkInfo.xml (entry: _c_int00_noargs). Modifying startup or linker scripts changes run behavior and memory map.

PR / edit guidance for agents
- If asked to run a build, confirm CCS/MSP SDK availability and exact install paths before attempting. If not available, state you cannot produce a valid binary.
- If editing C source, operate on the files in BSP/, Drive/, Menu/, and main.c. Small logical patches are fine; avoid touching generated files unless the change purpose is to update checked-in build artifacts and you understand the repercussions.

Quick grep targets when exploring
- main.c, Menu/*.c, Drive/OLED_Drv/OLED.c, BSP/delay.c
- .cproject (toolchain/config), Debug/optoelectronicrace_26_linkInfo.xml (actual linker command/flags)

If you need more information
- Ask whether CCS and the MSPM0 SDK are installed on the machine and where (paths are required to run make outside CCS).

Env vars & example values (what to match before running make)
- Variables referenced by .cproject that must resolve when building outside CCS: COM_TI_MSPM0_SDK_INSTALL_DIR, COM_TI_MSPM0_SDK_INCLUDE_PATH, COM_TI_MSPM0_SDK_LIBRARY_PATH, COM_TI_MSPM0_SDK_LIBRARIES, COM_TI_MSPM0_SDK_SYMBOLS, COM_TI_MSPM0_SDK_SYSCONFIG_MANIFEST, SYSCONFIG_TOOL_INCLUDE_PATH, SYSCONFIG_TOOL_LIBRARY_PATH, SYSCONFIG_TOOL_LIBRARIES, SYSCONFIG_TOOL_SYMBOLS, SYSCONFIG_TOOL_SYSCONFIG_MANIFEST, CG_TOOL_ROOT, PROJECT_ROOT, PROJECT_BUILD_DIR.
- Example values embedded in the generated Debug/link-info in this repository (use these only as a template — your machine's CCS/SDK may differ):

  - PROJECT_ROOT -> D:/MSP-SDK/CCS_project/optoelectronicrace_26
  - CG_TOOL_ROOT -> D:/CCS/ccs/tools/compiler/ti-cgt-armllvm_4.0.4.LTS
  - Linker / tool example: D:\\CCS\\ccs\\tools\\compiler\\ti-cgt-armllvm_4.0.4.LTS\\bin\\tiarmlnk (shown in Debug/optoelectronicrace_26_linkInfo.xml)
  - Project include paths (examples): D:/MSP-SDK/CCS_project/optoelectronicrace_26/BSP, D:/MSP-SDK/CCS_project/optoelectronicrace_26/Drive, D:/MSP-SDK/CCS_project/optoelectronicrace_26/Drive/OLED_Drv, D:/MSP-SDK/CCS_project/optoelectronicrace_26/Menu
- Before attempting a make-based build: verify CCS is installed and set those variables (or build from within CCS). If they don't match, the generated makefiles will reference nonexistent absolute paths and the build will fail.
